# 커뮤니티 그룹 기능 명세

## DB 스키마

### `members` (사용자 / 인증)

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 멤버 고유 ID |
| username | VARCHAR(50) UNIQUE | 로그인 아이디 |
| password_hash | VARCHAR(255) | bcrypt 해시 비밀번호 |
| name | VARCHAR(50) | 이름 |
| created_at | TIMESTAMP | 생성 시각 |

> **사용자 식별**: 세션(`req.session.memberId`)으로 관리. 로그인 시 서버에 세션 저장, 브라우저에 `connect.sid` 쿠키 발급.

### `groups`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 그룹 고유 ID |
| name | VARCHAR(100) | 그룹 이름 |
| code | CHAR(8) UNIQUE | 초대 코드 (대문자+숫자, 중복 없음) |
| max_members | INT DEFAULT 20 | 최대 인원 |
| created_at | TIMESTAMP | 생성 시각 |

### `group_members`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | - |
| group_id | INT FK → groups.id | 그룹 참조 (CASCADE DELETE) |
| member_id | INT FK → members.id | 멤버 참조 (CASCADE DELETE) |
| nickname | VARCHAR(50) | 해당 그룹 내 닉네임 (참여 시 입력) |
| is_creator | TINYINT(1) | 그룹 생성자 여부 |
| joined_at | TIMESTAMP | 참여 시각 |
| UNIQUE | (group_id, member_id) | 중복 참여 방지 |

## 인증 API

### `POST /api/auth/register`

회원가입

**Body**
```json
{
  "username": "mash_user",
  "password": "123456",
  "name": "홍길동"
}
```

**Response 201**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | username, password, name 중 누락 |
| 400 | 비밀번호 6자 미만 |
| 409 | 이미 사용 중인 아이디 |
| 500 | DB 오류 |

---

### `POST /api/auth/login`

로그인. 성공 시 세션 생성 및 쿠키(`connect.sid`) 발급.

**Body**
```json
{ "username": "mash_user", "password": "123456" }
```

**Response 200**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | username 또는 password 누락 |
| 401 | 아이디 또는 비밀번호 불일치 |
| 500 | DB 오류 |

---

### `POST /api/auth/logout`

로그아웃. 서버 세션 삭제.

**Response 200**
```json
{ "ok": true }
```

---

### `GET /api/auth/me`

현재 세션 확인.

**Response 200**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 401 | 미로그인 |

```json
  {
    "id": 1,
    "name": "Mash-Up 10기",

    "maxMembers": 20,
    "memberCount": 5,
    "isCreator": true
  }
]
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 401 | 미로그인 |
| 500 | DB 오류 |

---

### `POST /api/community/groups`

그룹 생성 (생성자 자동 참여)

**Body**
```json
{
  "name": "그룹 이름",
  "nickname": "그룹 내 닉네임",
  "maxMembers": 20
}
```

**Response 201**
```json
{ "groupId": 1, "name": "그룹 이름", "code": "KQ7RMP2A" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | name 또는 nickname 누락 |
| 401 | 미로그인 |
| 500 | DB 오류 |

---

### `GET /api/community/groups/verify`

초대 코드로 그룹 미리보기 (참여 전 확인용)

**Query**
```
code: string (8자리, 필수)
```

**Response 200**
```json
{ "id": 1, "name": "그룹 이름", "maxMembers": 20, "memberCount": 5 }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | code 누락 |
| 401 | 미로그인 |
| 404 | 존재하지 않는 초대 코드 |
| 500 | DB 오류 |

> 정원 초과 여부는 프론트에서 `memberCount >= maxMembers` 로 체크해 step 2 진입 차단.

**Body**

## 프론트엔드 기능

### 인증
- 커뮤니티 탭 진입 시 세션 확인 (`GET /api/auth/me`)
- 미로그인이면 로그인/회원가입 모달 자동 표시
- 로그인 성공 시 그룹 목록 자동 갱신
- 설정(⚙) 버튼 → 로그아웃 메뉴 → 확인 모달 → 세션 삭제

### 그룹 목록 화면
- 내가 속한 그룹 카드 목록 표시
- 각 카드: 그룹명, 현재 멤버 수 / 최대 멤버 수
- 헤더 뱃지: 참여 중인 그룹 수
- 커뮤니티 탭 클릭 또는 모달 닫힐 때 자동 갱신

### 그룹 생성
1. 그룹 이름 + 닉네임 입력
2. 생성 완료 → 초대 코드 표시 (복사 버튼 포함)
3. 모달 닫기(X, 바깥 클릭, 그룹 입장) → 목록 자동 갱신

### 그룹 참여
1. 초대 코드 입력 + 닉네임 입력
2. 코드 확인 → 그룹 미리보기 (그룹명, 인원) + 정원 초과 시 차단
3. 참여하기 → 성공 시 채팅 화면 진입

### 그룹 진입
- 진입 버튼 클릭 시 해당 그룹 멤버 수만큼 그리드 렌더링
- 멤버 수 초과 시 스크롤

### 그룹 카드 더보기(⋯)
- 초대 코드 보기: 해당 그룹의 8자리 코드 + 복사 버튼

### 그룹 나가기
- 커스텀 확인 모달 표시 후 나가기
- 나가기 성공 시 목록 자동 갱신 + 토스트

# 사용량(Usage) 탭 기능 명세

Claude Code의 JSONL 대화 로그를 파싱해 토큰 사용량을 로컬에서 집계하고 시각화하는 탭.

## DB 스키마 (SQLite · `data/usage.db`)

### `usage_records`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INTEGER PK AUTOINCREMENT | - |
| session_id | TEXT | Claude 세션 UUID |
| project_path | TEXT | 작업 디렉토리 경로 |
| model | TEXT | 정규화된 모델명 (예: `Haiku 4.5`) |
| input_tokens | INTEGER | 입력 토큰 수 |
| output_tokens | INTEGER | 출력 토큰 수 |
| cache_creation_tokens | INTEGER | 캐시 생성 토큰 수 |
| cache_read_tokens | INTEGER | 캐시 읽기 토큰 수 |
| recorded_at | TEXT | ISO 8601 타임스탬프 |

### `file_offsets`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| file_path | TEXT PK | JSONL 파일 절대 경로 |
| last_offset | INTEGER | 마지막으로 처리 완료한 바이트 위치 |
| last_synced_at | TEXT | 마지막 동기화 시각 |

> 증분 읽기의 핵심. `last_offset`은 레코드 삽입과 단일 트랜잭션으로 갱신되어 원자성 보장.

### `usage_sessions`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| session_id | TEXT PK | Claude 세션 UUID |
| session_name | TEXT | JSONL 첫 user 메시지 앞 20자 |

## 데이터 수집 구조

### JSONL 파일 위치

```
~/.claude/projects/{project-dir}/{session-id}.jsonl
~/.claude/projects/{project-dir}/{session-id}/subagents/{sub-session-id}.jsonl
```

### 수집 흐름

```
서버 시작
  └─ restoreFromSQLite()   DB → 메모리 상태 복구
  └─ initialScan()         전체 JSONL 스캔 (미처리 구간만)
  └─ startWatcher()        chokidar 감시 시작
  └─ startPeriodicRescan() 30초 주기 보완 스캔 시작

파일 변경 감지 (chokidar, awaitWriteFinish 300ms)
  └─ enqueueFileParse()    파일별 직렬 큐에 등록
       └─ parseFile()      Buffer 스트림으로 증분 읽기
            └─ 0x0A(\n) 기반 라인 분리 → JSON 파싱
            └─ assistant 타입만 usage 추출 → applyRecord() (메모리)
            └─ stmtFlushAndUpdateOffset() ─ 단일 트랜잭션
                 ├─ usage_records INSERT
                 └─ file_offsets UPDATE
```

> chokidar와 30초 주기 스캔 모두 동일한 `enqueueFileParse()` 큐를 통해 실행되므로 같은 파일의 동시 처리 없음.

### 파서 특이사항

- **Buffer 기반 스트림**: `encoding` 없이 읽어 `\n`(0x0A) 바이트로 라인 분리 → 한글·이모지 포함 라인에서도 오프셋 오계산 없음
- **증분 읽기**: `processedOffset = offset + totalBytesRead - remainder.length` 로 처리 완료된 마지막 완전한 라인의 끝 바이트만 기록
- **세션 이름**: offset=0인 첫 읽기에서만 `user` 타입 첫 메시지 앞 20자를 `usage_sessions`에 저장

## API

### `GET /api/usage/snapshot`

현재 `usageState` 전체를 JSON으로 반환. 탭 최초 진입 시 사용.

**Response 200**: `usageState` 객체

---

### `GET /api/usage/stream`

SSE(Server-Sent Events) 스트림. 연결 즉시 현재 상태를 전송하고, 이후 변경 시마다 `usage_update` 이벤트 발행.

| 이벤트명 | 발생 조건 |
| --- | --- |
| `connected` | 연결 수립 직후 |
| `usage_update` | 연결 직후 초기 데이터 / JSONL 파싱 완료 / 레이트 리밋 변경 |

## 프론트엔드 기능

### 요약 지표
- 주간 활성 세션 수 (최근 7일 내 활동 세션 합산)
- 레이트 리밋 사용률 배지 (5시간 / 7일 창)

### 주간 토큰 사용량 차트
- 최근 7일 일별 바 차트, 모델별 색상 구분
- 막대 hover 시 모델명·토큰 수 툴팁

### 프로젝트별 사용량 테이블
- 프로젝트 행: 총 토큰, 세션 수, 캐시 효율, 마지막 활동
- 아코디언 펼치기로 세션 목록 확인 (SSE 업데이트 시 열림 상태 유지)
- 현재 활성 Claude 세션이 있는 프로젝트/세션은 "실시간" 표시
  - 프로젝트 행: 해당 프로젝트 경로에 활성 세션이 1개 이상이면 표시
  - 세션 행: session_id 또는 세션 이름이 일치하는 활성 세션이 있을 때만 표시

### 데이터 지연 안내
- 로그 기반 지표는 파일 감시 주기(최대 30초) 지연이 있음을 UI에 안내

## 알려진 미완성 / 개선 필요 사항

| 항목 | 설명 |
| --- | --- |
| 메모리 풀 로드 | 서버 시작 시 `restoreFromSQLite()`가 `usage_records` 전체를 메모리에 적재. 데이터 누적 시 시작 시간 증가 및 메모리 소모. 개선 방향: `usageState`를 캐시로 전환하고, 집계 쿼리(`GROUP BY`, `SUM`)를 DB에서 직접 실행하여 결과 반환 |
| 최대 30초 지연 | chokidar 이벤트 누락 시 주기 스캔(30초)으로 보완하므로 최악의 경우 30초 지연 발생. 완전 실시간이 필요하다면 OpenTelemetry 기반 수집 구조로 전환 필요 |
