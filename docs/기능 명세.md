# 커뮤니티 그룹 기능 명세

## DB 스키마

### `members` (사용자 / 인증)

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 멤버 고유 ID |
| username | VARCHAR(50) UNIQUE | 로그인 아이디 |
| password_hash | VARCHAR(255) | bcrypt 해시 비밀번호 |
| name | VARCHAR(50) | 이름 |
| created_at | TIMESTAMP | 생성 시각 |

> **사용자 식별**: 세션(`req.session.memberId`)으로 관리. 로그인 시 서버에 세션 저장, 브라우저에 `connect.sid` 쿠키 발급.

### `groups`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 그룹 고유 ID |
| name | VARCHAR(100) | 그룹 이름 |
| code | CHAR(8) UNIQUE | 초대 코드 (대문자+숫자, 중복 없음) |
| max_members | INT DEFAULT 20 | 최대 인원 |
| created_at | TIMESTAMP | 생성 시각 |

### `group_members`

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | - |
| group_id | INT FK → groups.id | 그룹 참조 (CASCADE DELETE) |
| member_id | INT FK → members.id | 멤버 참조 (CASCADE DELETE) |
| nickname | VARCHAR(50) | 해당 그룹 내 닉네임 (참여 시 입력) |
| is_creator | TINYINT(1) | 그룹 생성자 여부 |
| joined_at | TIMESTAMP | 참여 시각 |
| UNIQUE | (group_id, member_id) | 중복 참여 방지 |

## 인증 API

### `POST /api/auth/register`

회원가입

**Body**
```json
{
  "username": "mash_user",
  "password": "123456",
  "name": "홍길동"
}
```

**Response 201**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | username, password, name 중 누락 |
| 400 | 비밀번호 6자 미만 |
| 409 | 이미 사용 중인 아이디 |
| 500 | DB 오류 |

---

### `POST /api/auth/login`

로그인. 성공 시 세션 생성 및 쿠키(`connect.sid`) 발급.

**Body**
```json
{ "username": "mash_user", "password": "123456" }
```

**Response 200**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | username 또는 password 누락 |
| 401 | 아이디 또는 비밀번호 불일치 |
| 500 | DB 오류 |

---

### `POST /api/auth/logout`

로그아웃. 서버 세션 삭제.

**Response 200**
```json
{ "ok": true }
```

---

### `GET /api/auth/me`

현재 세션 확인.

**Response 200**
```json
{ "memberId": 1, "username": "mash_user", "name": "홍길동" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 401 | 미로그인 |

---

## 커뮤니티 API

> 모든 커뮤니티 API는 로그인 필수. 미로그인 시 401 반환.

### `GET /api/community/groups`

내가 속한 그룹 목록 조회

**Response 200**
```json
[
  {
    "id": 1,
    "name": "Mash-Up 10기",
    "code": "KQ7RMP2A",
    "maxMembers": 20,
    "memberCount": 5,
    "isCreator": true
  }
]
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 401 | 미로그인 |
| 500 | DB 오류 |

---

### `POST /api/community/groups`

그룹 생성 (생성자 자동 참여)

**Body**
```json
{
  "name": "그룹 이름",
  "nickname": "그룹 내 닉네임",
  "maxMembers": 20
}
```

**Response 201**
```json
{ "groupId": 1, "name": "그룹 이름", "code": "KQ7RMP2A" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | name 또는 nickname 누락 |
| 401 | 미로그인 |
| 500 | DB 오류 |

---

### `GET /api/community/groups/verify`

초대 코드로 그룹 미리보기 (참여 전 확인용)

**Query**
```
code: string (8자리, 필수)
```

**Response 200**
```json
{ "id": 1, "name": "그룹 이름", "maxMembers": 20, "memberCount": 5 }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | code 누락 |
| 401 | 미로그인 |
| 404 | 존재하지 않는 초대 코드 |
| 500 | DB 오류 |

> 정원 초과 여부는 프론트에서 `memberCount >= maxMembers` 로 체크해 step 2 진입 차단.

---

### `POST /api/community/groups/join`

초대 코드로 그룹 참여

**Body**
```json
{ "code": "KQ7RMP2A", "nickname": "그룹 내 닉네임" }
```

**Response 201**
```json
{ "groupId": 1, "name": "그룹 이름" }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 400 | code 또는 nickname 누락 |
| 401 | 미로그인 |
| 404 | 존재하지 않는 초대 코드 |
| 409 | 정원 초과 |
| 409 | 이미 참여 중인 그룹 |
| 500 | DB 오류 |

---

### `DELETE /api/community/groups/:groupId/leave`

그룹 나가기. 마지막 멤버가 나가면 그룹 자동 삭제.

**Response 200**
```json
{ "ok": true }
```

**에러**

| 상태 | 사유 |
| --- | --- |
| 401 | 미로그인 |
| 404 | 해당 그룹의 멤버가 아님 |
| 500 | DB 오류 |

## 프론트엔드 기능

### 인증
- 커뮤니티 탭 진입 시 세션 확인 (`GET /api/auth/me`)
- 미로그인이면 로그인/회원가입 모달 자동 표시
- 로그인 성공 시 그룹 목록 자동 갱신
- 설정(⚙) 버튼 → 로그아웃 메뉴 → 확인 모달 → 세션 삭제

### 그룹 목록 화면
- 내가 속한 그룹 카드 목록 표시
- 각 카드: 그룹명, 현재 멤버 수 / 최대 멤버 수
- 헤더 뱃지: 참여 중인 그룹 수
- 커뮤니티 탭 클릭 또는 모달 닫힐 때 자동 갱신

### 그룹 생성
1. 그룹 이름 + 닉네임 입력
2. 생성 완료 → 초대 코드 표시 (복사 버튼 포함)
3. 모달 닫기(X, 바깥 클릭, 그룹 입장) → 목록 자동 갱신

### 그룹 참여
1. 초대 코드 입력 + 닉네임 입력
2. 코드 확인 → 그룹 미리보기 (그룹명, 인원) + 정원 초과 시 차단
3. 참여하기 → 성공 시 채팅 화면 진입

### 그룹 진입
- 진입 버튼 클릭 시 해당 그룹 멤버 수만큼 그리드 렌더링
- 멤버 수 초과 시 스크롤

### 그룹 카드 더보기(⋯)
- 초대 코드 보기: 해당 그룹의 8자리 코드 + 복사 버튼

### 그룹 나가기
- 커스텀 확인 모달 표시 후 나가기
- 나가기 성공 시 목록 자동 갱신 + 토스트

## 에러 핸들링

### 토스트 알림 (모달 위 z-index 300)

| 상황 | 타입 | 동작 |
| --- | --- | --- |
| 코드 확인 실패 | error | 토스트, 모달 유지 (재입력 가능) |
| 그룹 참여 실패 | error | 토스트, 모달 닫힘 |
| 그룹 생성 실패 | error | 토스트, 모달 유지 (재시도 가능) |
| 그룹 나가기 실패 | error | 토스트 |
| 그룹 나가기 성공 | info | 토스트 |

## 알려진 미완성 / 개선 필요 사항

| 항목 | 설명 |
| --- | --- |
| 세션 저장소 | 현재 서버 메모리에 저장 → 서버 재시작 시 세션 초기화. 운영 환경에서는 DB/Redis 저장 필요 |
| 클립보드 복사 실패 | non-HTTPS 환경에서 조용히 실패 |
| 그룹 진입 컨텍스트 | 진입 시 그룹 이름/내 닉네임이 채팅 헤더에 표시 안 됨 |
